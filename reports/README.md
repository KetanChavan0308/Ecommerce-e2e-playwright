# Test Reports Directory

Industry-standard test reporting structure for the Ecommerce E2E Playwright Automation Framework.

## Directory Structure

```
reports/
├── README.md                 # This file - documentation
├── html/                     # HTML test reports
│   ├── index.html           # Main HTML report
│   ├── data/                # Report data files
│   └── trace/               # Trace viewer
├── xml/                      # JUnit XML reports (CI/CD integration)
├── test-results/            # Detailed test results
├── summaries/               # Executive summary reports
└── archive/                 # Historical reports
```

## Report Types

### 1. HTML Reports (`html/`)
- **Purpose**: Interactive, human-readable test execution reports
- **Generated by**: Playwright test runner
- **Contains**: Test duration, pass/fail status, screenshots, traces
- **Access**: Open `index.html` in a web browser

### 2. XML Reports (`xml/`)
- **Purpose**: Machine-readable JUnit format for CI/CD pipelines
- **Generated by**: Test framework
- **Integration**: Jenkins, GitHub Actions, Azure DevOps, etc.
- **Format**: JUnit XML standard

### 3. Test Results (`test-results/`)
- **Purpose**: Detailed execution logs and artifacts
- **Contains**: Videos, screenshots, traces, logs
- **Retention**: Typically archived after analysis

### 4. Summaries (`summaries/`)
- **Purpose**: Executive summary reports for stakeholders
- **Format**: Markdown, JSON, or PDF
- **Contents**: Pass rate, execution time, trend analysis

### 5. Archive (`archive/`)
- **Purpose**: Historical storage of previous test runs
- **Organization**: By date (YYYY-MM-DD folder structure)
- **Retention**: As per company policy

## Generating Reports

### Run All Tests and Generate Reports
```bash
npx playwright test
```

### Run Specific Test Suite
```bash
npx playwright test tests/dataDrivanTesting.spec.ts
```

### View HTML Report
```bash
npx playwright show-report
```

## CI/CD Integration

### GitHub Actions Example
Reports are automatically generated and can be uploaded as artifacts:

```yaml
- name: Upload test results
  if: always()
  uses: actions/upload-artifact@v3
  with:
    name: playwright-report
    path: reports/
    retention-days: 14
```

### Jenkins Integration
JUnit XML reports can be published using:
```xml
<junit testResults="reports/xml/*.xml" />
```

## Report Retention Policy

- **Latest Reports**: Kept in active directories
- **Weekly Archive**: Automated archival to `archive/` folder
- **Monthly Cleanup**: Reports older than 90 days moved to long-term storage
- **Retention**: Minimum 6 months for compliance

## Best Practices

1. **Run tests regularly**: Maintain continuous test execution
2. **Analyze failures**: Review failed tests immediately
3. **Track trends**: Monitor pass rate over time
4. **Archive reports**: Move old reports to archive directory
5. **Share summaries**: Communicate results to stakeholders

## Report Status Codes

| Status | Description |
|--------|-------------|
| ✅ PASS | Test execution successful |
| ❌ FAIL | Test assertion failed |
| ⏭️ SKIP | Test skipped (dependencies/conditions not met) |
| ⚠️ FLAKY | Test passed but has intermittent failures |
| 🔄 TIMEOUT | Test exceeded execution time limit |

## Accessing Reports

1. **Local Machine**: Run `npm run report` (if configured in package.json)
2. **CI/CD Pipeline**: Download from artifact storage
3. **Web Server**: Host reports on internal server for team access

## Related Files

- `playwright.config.ts` - Playwright configuration with report settings
- `package.json` - NPM scripts for running tests
