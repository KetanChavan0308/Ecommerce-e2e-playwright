## Report Types

### 1. HTML Reports (`html/`)

- **Purpose**: Interactive, human-readable test execution reports
- **Generated by**: Playwright HTML Reporter
- **Contains**:
  - Test duration & execution time
  - Pass/fail status with error messages
  - Screenshots (captured on failure only)
  - Full Trace Viewer with step-by-step recording
- **Access**: Run `npx playwright show-report` or open `index.html` in browser
- **Artifacts Included**:
  - Trace files (`.webm` format)
  - Screenshots on failure
  - Network logs
  - Console logs

### 2. XML Reports (`xml/`)

- **Purpose**: JUnit XML format for CI/CD integration
- **Generated by**: Playwright JUnit Reporter
- **Output File**: `reports/results.xml`
- **Integration**: GitHub Actions, Jenkins, Azure DevOps, GitLab CI
- **Contains**: Test results, durations, failure details
- **Use Case**: Automated report parsing in CI/CD pipelines

### 3. Allure Reports (`allure-results/`)

- **Purpose**: Advanced, detailed test reporting with analytics
- **Generated by**: allure-playwright plugin
- **Contains**:
  - Test execution timeline
  - Failure root cause analysis
  - Test history & trend analysis
  - Artifact attachments
  - Environment information
- **Generation**: `npx allure serve reports/allure-results`
- **Features**: Beautiful visuals, historical tracking, failure categorization

### 4. Test Results Artifacts (`test-results/`)

- **Purpose**: Detailed execution logs and recordings
- **Contains**:
  - Videos (`.webm` format) - retained on failure
  - Screenshots (PNG) - captured on assertion failure
  - Traces (`.zip`) - recorded on first retry
  - Console logs
  - Network HAR files
- **Retention**: Automatically managed by Playwright
- **Organization**: Folder per test case with unique identifiers

### 5. Summaries (`summaries/`)

- **Purpose**: Executive summary reports for stakeholders
- **Format**: Markdown, JSON, or PDF
- **Contents**:
  - Total pass/fail counts
  - Pass rate percentage
  - Execution time summary
  - Failed test list with reasons
  - Trend analysis
- **Usage**: Share with team & management

### 6. Archive (`archive/`)

- **Purpose**: Historical storage of previous test runs
- **Organization**: By date (YYYY-MM-DD or build ID)
- **Structure**:
  ```
  archive/
  ├── 2026-04-15/
  │   ├── run-001/
  │   └── run-002/
  ├── 2026-04-16/
  │   └── run-001/
  ```
- **Retention**: Configurable retention policy
- **Use Case**: Historical analysis, regression tracking

## Generating Reports

### Run All Tests and Generate Reports

```bash
npx playwright test
```

Executes all test suites across all browsers defined in configuration.

### Run Specific Test Suite

```bash
# Login tests only
npx playwright test tests/Login.spec.ts

# Data-driven tests only (14 test cases)
npx playwright test tests/dataDrivanTesting.spec.ts

# Purchase order tests only
npx playwright test tests/purchaseOrder.spec.ts

# Account registration tests only
npx playwright test tests/AccountRegistration.spec.ts
```

### Run Tests with Options

```bash
# Run tests in headed mode (see browser window)
npx playwright test --headed

# Run tests in debug mode
npx playwright test --debug

# Run specific test by name
npx playwright test -g "Login Test: Email Not Registered"

# Run with trace enabled
npx playwright test --trace on
```

### View Generated Reports

```bash
# View HTML Report
npx playwright show-report

# View Allure Report (if allure-playwright installed)
npx allure serve reports/allure-results
```

## CI/CD Integration

### GitHub Actions (Configured)

Example workflow file already created at `.github/workflows/playwright.yml`:

```yaml
name: Playwright Tests
on:
  push:
    branches: [main, master]
  pull_request:
    branches: [main, master]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with:
          node-version: 18
      - run: npm ci
      - run: npx playwright install
      - run: npx playwright test
      - name: Upload test results
        if: always()
        uses: actions/upload-artifact@v3
        with:
          name: playwright-report
          path: reports/
          retention-days: 14
```

### Jenkins Integration

Configure Jenkins pipeline with:

```groovy
pipeline {
    agent any
    stages {
        stage('Install') {
            steps {
                sh 'npm ci && npx playwright install'
            }
        }
        stage('Test') {
            steps {
                sh 'npx playwright test'
            }
        }
    }
    post {
        always {
            junit 'reports/results.xml'
            publishHTML([
                reportDir: 'reports/html',
                reportFiles: 'index.html',
                reportName: 'Playwright Report'
            ])
        }
    }
}
```

### Azure DevOps

```yaml
trigger:
  - main

pool:
  vmImage: "ubuntu-latest"

steps:
  - task: NodeTool@0

```

## Troubleshooting

### Report Not Generated

1. **Check playwright.config.ts** - Ensure reporters are configured
2. **Verify output directory** - `outputDir: "reports/test-results"` exists
3. **Run with verbose logging** - `npx playwright test --reporter=verbose`

### Browser Installation Issues

```bash
# Reinstall all browsers
npx playwright install

# Install with dependencies
npx playwright install --with-deps
```

### Trace Viewer Not Loading

- Ensure trace files are generated: Check `use: { trace: "on-first-retry" }` in config
- View trace locally: `npx playwright show-trace trace.zip`

### Allure Report Empty

```bash
# Ensure plugin is installed
npm install --save-dev allure-playwright

# Generate report
npx allure serve reports/allure-results
```
